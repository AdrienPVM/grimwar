# Content sources

This folder holds the raw input files for the content pipeline. Everything inside `aidedd/`, `pdfs/`, and `extracted/` is **gitignored** — it stays on Adrien's machine.

## What to drop where

### `aidedd/` — HTML files scraped from AideDD

One `.html` file per entity. Filename convention helps the parser detect the type:

```
aidedd/
├── sorts/
│   ├── boule-de-feu.html
│   ├── lumiere.html
│   └── ...
├── monstres/
│   ├── dragon-rouge-adulte.html
│   └── ...
├── objets-magiques/
│   └── ...
├── classes/
│   └── magicien.html
├── races/
│   └── demi-elfe.html
└── historiques/
    └── sage.html
```

You can also dump them flat — the parser will inspect content to detect the type. But grouped folders make debugging easier.

### `pdfs/` — Official PDFs

Put each PDF here:

```
pdfs/
├── srd-5.1.pdf                  # SRD 5.1 (legally redistributable)
├── free-rules-2024.pdf          # Free Rules 2024 (legally redistributable)
└── dmg.pdf                      # Dungeon Master's Guide (private, not for redistribution)
```

**Important**:
- `srd-5.1.pdf` and `free-rules-2024.pdf` → extracted content goes into `public/data/` (committed, shipped publicly).
- `dmg.pdf` → extracted content goes into `content-sources/extracted/dmg/` (gitignored) and gets uploaded to **your private Firestore collection** only. It's never bundled in the app.

If you add other PDFs (PHB, MM, etc.), the parser will need new mappings. Either teach `scripts/parse-srd-text.ts` or upload them as private content like DMG.

### `extracted/` — Pipeline output

This folder is auto-generated by `scripts/*.ts`. Don't edit by hand.

```
extracted/
├── raw/                         # plain text from PDFs (intermediate)
├── aidedd/                      # parsed AideDD JSON (intermediate)
├── srd/                         # parsed SRD JSON (intermediate, becomes public/data)
├── free-rules/                  # parsed Free Rules JSON (intermediate)
└── dmg/                         # parsed DMG JSON (private — uploaded to Firestore)
```

## Running the pipeline

In order:

```bash
# 1. Extract raw text from all PDFs
pnpm content:extract-pdf

# 2. Parse the AideDD HTML files
pnpm content:parse-aidedd

# 3. Parse the SRD / Free Rules text
pnpm content:parse-srd

# 4. Merge everything into public/data/*.json (validated against Zod schemas)
pnpm content:build

# 5. Upload private DMG content to your Firestore (requires service account key)
pnpm content:upload-private
```

For details, see `plans/04-content-pipeline.md`.

## Legal note

- **SRD 5.1** and **Free Rules 2024** content is freely redistributable under the OGL / WotC's free distribution license. Parsed output goes into `public/data/` and ships with the app.
- **DMG / PHB / MM** content is **copyrighted by Wizards of the Coast**. For personal use only. The pipeline keeps it strictly in your private Firestore collection (`users/{your_uid}/customContent/`) — accessible to you when you're logged in, but never bundled, never published.
- **AideDD translations**: the French translations are AideDD's editorial work. They map onto SRD entity IDs in the public data, so what ships is "an SRD entity, with the AideDD translation as one of its names". If AideDD's terms object to this, the pipeline can be adapted to use machine translations or no translations.

When in doubt, keep content private.
