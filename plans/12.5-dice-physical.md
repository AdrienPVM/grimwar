# Plan 12.5 — Dice mode (physical)

## Goal

Couche **mode physique** par-dessus le moteur digital de plan 12. L'app indique quoi lancer IRL (« Lance 1d20 — Attaque à l'épée longue »), le joueur saisit la/les **face(s) brute(s)** ; l'app applique les modificateurs, détecte nat 20/1, gère avantage/désavantage, calcule le total. La saisie est **optionnelle** : le joueur peut **« Passer »** et fermer sans rien logger (le slot/coût de sort, déjà consommé au moment du cast, reste consommé — c'est le chemin intentionnel et non-erreur).

Le pivot `rollWithFlags` devient mode-aware : il branche sur `effectiveDiceMode(user, campaign)` (campagne = `null` en S1). Le retour devient `RollResult | null` (null si le joueur « Passe »). Les 8 call sites de la fiche ajoutent un null-guard d'une ligne. La séquence attaque → dégâts est encapsulée dans `rollAttackDamage` qui gère le gate **Touché / Raté** (auto-crit si face 20, auto-raté si face 1, sinon choix manuel) et le prompt de dégâts **avec dés doublés en cas de crit** (modificateur non doublé, conforme SRD 5e).

Un toggle minimal **Digital / Physique** est exposé dans le header du `<RollHistoryPanel />` (le panel est déjà ouvrable depuis la fiche dès plan 12, étape 24). Plan 35 (S5) déplacera ce toggle dans le full settings screen + ajoutera `followCampaignDiceMode`.

**Hors scope** : le hand-off MJ (réception du total de dégâts physique côté DM, sélection de cible, application) — implémenté plan 24 (encounters) car il dépend du combat tracker + de l'existence d'une campagne (S2+).
