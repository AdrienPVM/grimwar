# Plan 13.14 — PROTOTYPE Layout desktop fiche (v0)

> **STATUT** : PROTOTYPE labellisé livré 2026-05-26 — Layout desktop pour `SheetScreen`, à arbitrer par Adrien avant v1 (densification multi-col des cards).

## Goal

Donner à la fiche de personnage (`/character/:id`) une **première coquille desktop** qui exploite le viewport ≥ 1024 px au lieu de rester cantonnée à un rail centré 420 px (~75 % de dead space identifié comme **🔴 BLOQUANT** dans l'audit 13.14-RESPONSIVE-AUDIT.md).

C'est un v0 délibérément **minimal et non-invasif** : la coquille passe en 2 colonnes (sidebar 280 px + main fluide), mais les sub-cards internes (HpMegaCard, AttacksList, MagicCircle, SpellList, etc.) restent en `max-w-[420px]` historique. La densification multi-col des cards (xl: grid 2-col) est différée à v1 après arbitrage.

## Décision de design v0

### Coquille `sheet-screen.tsx`

Mobile / tablet (< `lg:` / < 1024 px) : **passthrough byte-identique**. Aucune régression. Les enfants gardent `mx-auto max-w-[420px]`.

`lg:` (≥ 1024 px) :
- Wrapper : `lg:mx-auto lg:grid lg:max-w-[1200px] lg:grid-cols-[280px_minmax(0,1fr)] lg:gap-8 lg:px-6 lg:pt-2`
- **Sidebar sticky 280 px** (`.sheet-desktop-aside`) — contient HeroCard + StatusStrip + ModeTabs. `lg:sticky lg:top-2 lg:self-start lg:max-h-[calc(100vh-1rem)] lg:overflow-y-auto`. Reste en place pendant que la zone main scrolle.
- **Zone main fluide** (`.sheet-desktop-main`) — contient le mode actif. `lg:min-w-0 lg:pt-2`.

### Utility CSS scopée (`globals.css`)

Pour éviter de toucher les **14+ sub-cards** qui portent toutes `mx-auto max-w-[420px]` (héritage mobile-first), on neutralise au sein de `.sheet-desktop-aside` et `.sheet-desktop-main`, au breakpoint `lg+` uniquement :

```css
@media (min-width: 1024px) {
  .sheet-state .sheet-desktop-aside > * {
    max-width: none;
    margin-left: 0;
    margin-right: 0;
    padding-left: 0;
    padding-right: 0;
  }
  .sheet-state .sheet-desktop-main > section {
    margin-left: 0;
    margin-right: 0;
  }
}
```

**Conséquence v0** :
- Le HeroCard, le StatusStrip et les ModeTabs s'asseyent sur 280 px (au lieu de `max-w-[420px] mx-auto`) — ils respectent la largeur sidebar.
- Le mode actif (Combat ou Magie) garde ses cards 420 px-larges mais alignées à GAUCHE dans la zone main au lieu d'être centrées avec du dead space des deux côtés.
- On gagne **immédiatement** ~280-400 px de respiration à 1440 px et la fiche cesse d'être un rail.

### Modes Combat + Magie

**Pas de refactor interne v0.** Les modes gardent leur container `mx-auto max-w-[420px] flex-col gap-3 px-4`. La règle CSS scopée les pousse en flux à gauche dans la main area.

## Décisions à arbitrer (v1)

| ID | Question | Convention v0 (par défaut) |
|---|---|---|
| **DT1** | Multi-col des cards à `xl:` (≥1280 px) ? | Non — cards 420 px en single col aligné à gauche. v1 pourrait passer en `xl: grid xl: grid-cols-2` pour HpMegaCard + AttacksList side-by-side. |
| **DT2** | Sidebar collapsible (toggle bouton) ? | Non — sidebar toujours visible à `lg+`. v1 pourrait ajouter un bouton « pleine largeur » pour le mode focus combat. |

| **DT5** | Status strip 4 cellules dans 280 px (PV / CA / Init / Vitesse) ? | Oui — chaque cellule ~60 px de large, ça tient. v1 pourrait passer en 2×2 si trop serré. |

| **DT7** | Largeur max wrapper desktop ? | `max-w-[1200px]` — adapté 1440 px, laisse marges latérales. v1 pourrait s'étendre à `max-w-[1440px]` ou plus pour 4K. |
