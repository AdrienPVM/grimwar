## 🔴 Bloquant — Défaut 1 : `SheetScreen` cantonnée à ~400px de largeur sur tablette + desktop

**Capture(s) clé(s)** :

- `04-sheet-combat-desktop-1440.png`
- `08-sheet-magie-desktop-1440.png`
- `12-sheet-essence-desktop-1440.png`
- `16-sheet-avoir-desktop-1440.png`
- (idem `03-...-tablet-1024.png` pour chaque mode)

**Symptôme** : à 1024px et 1440px, l'intégralité du contenu de la fiche
(hero card, HUD stats, mode tabs, panneaux) reste contraint à ~390-420px
de large, centré au milieu de l'écran. Sur desktop 1440, ~75 % de la
largeur de viewport est du fond noir/aurora décoratif. Le mode Avoir,
particulièrement à 1024+, montre une liste d'inventaire qui ne profite
pas du tout de la largeur disponible — chaque ligne pourrait tenir 3-4
colonnes en grille.

**Localisation probable** :

- `src/features/sheet/sheet-screen.tsx` — wrapper racine de la fiche, max-width inéquivalent.
- `src/features/sheet/modes/*-mode.tsx` — les modes héritent du wrapping.
- `src/features/sheet/hero/hero-card.tsx` — la hero card est probablement la première candidate à un breakpoint `lg:`.

**Note importante** : ce défaut **EST DÉJÀ ownerisé** par
`plans/13.14-sheet-responsive-desktop.md` (plan ouvert mais non livré). Cet
audit confirme la nécessité et délimite le périmètre :

- Modes les plus critiques pour le desktop : **Avoir** (liste, idéale 2-3
  colonnes) et **Essence** (l'hexagramme + skills list, idéale layout
  flex 2-col avec skills à droite).
- Modes intermédiaires : **Combat** (HUD + actions, layout 2-col HUD ↔
  cards), **Magie** (Cercle d'incantation + sorts préparés/connus en
  2-col).

**Fix suggéré (esprit, à arbitrer)** :

- Breakpoint `lg:` (~1024px) qui passe la fiche en 2-col (rail gauche
  hero + HUD + mode tabs / panneau droit modes content).
- Breakpoint `xl:` (~1280px) optionnel pour 3-col (rail gauche / mode content / rail droit événements ou inspecteur — futur).
- Préserver mobile-first : les changements de layout SONT additifs, pas
  réécrits.

**Sévérité** : 🔴 BLOQUANT pour le branding « PWA mobile-first cinematic »
DM-au-jeu sur ipad/desktop (ce qui sera l'usage du MJ Adrien lui-même
à terme).

## 🟡 Mineur — Défaut 2 : Wizard 2-col absent à 768px (passe en 2-col seulement à 1024+)

**Capture(s) clé(s)** :

- `02-wizard-landing-tablet-sm-768.png` (1-col stacké, TOC au-dessus du form)
- `03-wizard-landing-tablet-1024.png` (2-col TOC + form OK)

**Symptôme** : entre 768 et 1023px (iPad portrait), le wizard est en
layout 1-col stacké comme mobile. La TOC verticale (Identité / Classe /
Ascendance / …) prend toute la largeur en haut puis le form en dessous.
Visuellement aéré sur cette tranche d'écran, mais sous-utilisé.

**Localisation probable** :

- `src/features/wizard/wizard-screen.tsx` (ou son layout container) — le
  breakpoint actuel est probablement `lg:` (1024px) ; pourrait être `md:`
  (768px).

**Fix suggéré** : descendre le breakpoint 2-col du wizard à `md:` (768px).
Risque minimal — la TOC fait ~150px, le form ~600px → tient à 768.

**Sévérité** : 🟡 MINEUR (UX dégradée mais fonctionnelle ; iPad portrait
n'est pas la première cible).

## 🟡 Mineur — Défaut 3 : Barre de progression wizard très maigre à 1440px

**Capture(s) clé(s)** :

- `04-wizard-landing-desktop-1440.png`

**Symptôme** : la barre de progression « ÉTAPE 1 / 8 — IDENTITÉ » au-dessus
du card wizard est rendue avec un trait fin sur ~70 % de la largeur du
viewport. Sur desktop large c'est visuellement maigre (~2-3px de hauteur
contre ~1300px de largeur). À l'inverse, sur mobile elle est correctement
proportionnée.

**Localisation probable** :

- `src/features/wizard/wizard-screen.tsx` — la barre de progression
  utilise probablement `h-px` ou `h-0.5` sans breakpoint d'épaississement.

**Fix suggéré** : `h-px md:h-0.5 lg:h-1` (ou équivalent token design system)
pour épaissir progressivement à l'échelle.

**Sévérité** : 🟡 MINEUR (esthétique, pas fonctionnel).

## 🟢 Nitpick — Défaut 4 : Negative space cosmétique aurora desktop sur les écrans non-fiche

**Capture(s) clé(s)** :

- `04-wizard-landing-desktop-1440.png`

**Symptôme** : à 1440px, le card wizard est ~800px de large centré sur
1440px. Le reste est noir + aurora très subtil. Le card est correctement
proportionné en lui-même, mais l'aurora background pourrait porter plus
de texture/décor (manuscrit illuminé) pour habiller le vide. Strictement
cosmétique.

**Sévérité** : 🟢 NITPICK.
