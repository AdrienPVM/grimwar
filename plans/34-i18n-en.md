# Plan 31 — i18n EN

## Goal
Add English support. All UI strings translated. All entity JSON has `en` filled. Locale switcher in settings. Default for non-FR browsers becomes EN.

## Context
Read `docs/I18N.md` in full.

## Prerequisites
Sprint 4 complete.

## Steps
- [ ] 1. Audit `STRINGS` in `src/shared/lib/i18n.ts`: every key has FR; ensure no hardcoded strings remain in components (grep for hardcoded French in `.tsx`).
- [ ] 2. Add `en` translations for every key in `STRINGS`. Have Adrien review.
- [ ] 3. Audit `public/data/*.json`: every entity has `name.en` (most should already from PDF parsing; verify spot-checks).
- [ ] 4. Fill any missing `en` in entity prose. Use the PDF source's English text where possible.
- [ ] 5. Detect locale at sign-up:
    - `navigator.language` starts with `'fr'` → 'fr'
    - else → 'en'
    - User can override anytime in settings.
- [ ] 6. Settings → Language switcher: 2-option toggle (Français / English). Persisted in `users/{uid}.locale`.
- [ ] 7. Real-time locale switch: change updates Zustand slice, all `t()` / `localize()` re-render the tree.
- [ ] 8. Make sure `Intl.DateTimeFormat` and `Intl.NumberFormat` use the current locale via `format.ts` helpers.
- [ ] 9. Tests: e2e switch FR ↔ EN, verify all visible strings change appropriately on 3 representative screens (library, sheet combat, campaign settings).

### Final
- [ ] 10. `pnpm typecheck && pnpm test && pnpm lint`
- [ ] 11. Commit: `feat(i18n): English support (plan 31)`

## Definition of Done
- [ ] All UI strings localized
- [ ] All entity JSON has EN
- [ ] Locale switcher works without reload
- [ ] Date/number formatting locale-aware
- [ ] No hardcoded strings remain
