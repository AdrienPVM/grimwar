## Prémisses corrigées (enquête disque 2026-05-20)

Le stub reposait sur 5 hypothèses fausses, toutes vérifiées sur disque :

1. **« Cleanup = transformer le bundle existant »** → faux comme base. Les 330 entrées actuelles sont **d'origine AideDD** (provenance interdite par la politique de contenu lockée). Empreinte directe : **les 330 entrées ont `name.en = null`** alors que tous les bundles siblings (feats, ancestries, invocations) sont bilingues `fr`+`en`. Garder ces entrées = conserver 100 % d'une source interdite. Décision : **régénérer**, pas transformer.
2. **« `extract-srd-spells.ts` parse les PDF au runtime »** → le script **n'existe pas**, et aucun sibling ne parse de PDF : tous importent un module TS hand-curated `scripts/data/srd-*.ts` puis écrivent le JSON. Le pattern canonique est TS→JSON, pas PDF→JSON.
3. **« Les sorts sont SRD bilingues »** → faux : seuls les sorts sont FR-only (`en=null`), les autres bundles sont bilingues. Les sorts sont la seule exception et la signature de leur provenance AideDD.
4. **« D9 = ajouter 2 sorts manquants »** → faux. False Life est **déjà dans le bundle** sous l'id `simulacre-de-vie` ; la donnée d'ascendance Tieffelin pointe sur un **slug fantôme** `feinte-vie` qui ne résout nulle part. Seul **Ray of Sickness** est réellement absent — et l'ascendance Tieffelin Abyssale le référence sous le slug fantôme `rayon-de-maladie` tandis que l'audit D.3 le nomme « rayon empoisonné ». D9 = **corriger 2 slugs fantômes** + **ajouter 1 sort réellement manquant** sous son nom officiel FR vérifié.
5. **« DoD grep `source: "phb-2014"` »** → faux. Le champ `source` ne contient que `srd-5.2.1` (319) + `basic-rules` (11) ; aucun `phb-2014`. Le naming PHB 2014 est dans les **ids/noms FR**, pas dans `source`. Les assertions DoD du stub étaient périmées.
