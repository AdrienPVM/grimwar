## Contexte

Découvertes par le runner combinatoire du plan 13.12 (3 dettes tracées dans `plans/DEBT.md`), puis affinées par l'audit transversal d'ouverture de ce mini-plan :

- **D18** — trou de **contenu**. `buildAncestrySpellIds()` n'injecte que le triplet de lignage/héritage (cantrip + L3 + L5). Les sorts de **traits communs** (« Présence d'outre-monde » Tieffelin, « Gnome des forêts ») ne sont jamais poussés dans `knownSpells.ancestry`.
- **D19** — trou de **moteur**. Le style `defense` (+1 CA en armure) est sélectionnable au wizard mais n'écrit aucun champ dérivé.
- **D20** (nouvelle, découverte ici) — trou d'**affichage**. `computeAcFromArmor` / `acFromArmor` existent (`use-inventory-derived.ts:135`, `inventory-rules.ts:55`) mais **aucun consommateur UI** (grep : zéro). `StatusStrip` affiche `character.ac` = `10 + modDEX` désarmé (posé au build, `submit-from-wizard.ts:322`). Équiper une cotte de mailles ne change pas la CA affichée. D19 et D20 sont **techniquement couplées** (impossible d'appliquer un +1 conditionnel à l'armure sans d'abord câbler la CA d'armure) → résolues ensemble.

## Décisions (validées par Adrien avant écriture)

- **A2** (vs A1) — CA correctement câblée plutôt que +1 inconditionnel sur `character.ac`. A1 graverait une approximation SRD-fausse dans le champ canonique (Defense est conditionnel à l'armure ; rien n'est équipé au build). A2 résout D19 **et** le D20 pré-existant d'un coup. `character.ac` stocké reste `10+modDEX` (CA désarmée) ; la CA d'armure devient une **dérivation** consommée par l'UI. → l'invariant matrice `baseAc === 10 + modDEX` **reste vert** (pas de rouge-avant-vert sur `character.ac`) ; le rouge-avant-vert porte sur la **nouvelle dérivation de CA affichée**.
- **B1** (vs B2) — thaumaturgie **data-driven** : nouveau champ `commonSpellIds[]` sur le type `Ancestry`, édité dans `scripts/data/srd-ancestries-l1.ts`, régénéré dans `public/data/ancestries.json`. Respecte la politique gravée (contenu dans `scripts/data/`, paths protégés via PR — flow 13.13). B2 (slug en dur côté code) contournerait le mécanisme installé hier pour 5 min de gain.
- **Gnome forêts : fix présence maintenant (D18), cast différé (D12).** `communication-avec-les-animaux` (rituel, PB×/repos) → sa *présence* dans `knownSpells.ancestry` relève de D18 ; son *cast* à usage limité relève de D12 (déjà ouverte).

**Prémisses spéculatives écartées par l'audit :**
- Gnome « Présence inattendue » → **n'existe pas** dans le SRD CC (lignages = forêts/roches uniquement).
- Drakéide sorts de traits → **aucun** (souffle = action, vol = aptitude).
- Lignes raw FR `4645` (Druide), `6389` (Magicien), `8021`/`8454` (Paladin/Rôdeur) « toujours préparé » = **traits de classe**, pas d'ascendance → hors périmètre.

**Conclusion : exactement 2 injections** — Tiefling `thaumaturgie`, Gnome forêts `communication-avec-les-animaux`.
