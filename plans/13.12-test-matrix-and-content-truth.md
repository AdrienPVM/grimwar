## Volet structuré — Vérité du contenu (6 catégories)

Chaque catégorie devient une **suite de helpers réutilisables** posés dans `tests/helpers/content-truth/` :

### 1. Cohérence référentielle étendue
- Étend `tests/content-referential-integrity.test.ts` à toutes les références cross-bundle (skills, feats, invocations, languages, classes, sous-classes).
- Détecte aussi les entrées **orphelines** d'un bundle (slug sans aucun référent).

### 2. Contenu affiché = contenu attendu (identité)
- Helper `expectIdentityRender(bundleEntry, renderedNode)` qui assert titre + sous-titre + meta + description du nœud rendu vs entrée bundle exacte.
- À appliquer sur toutes les modales de détail (sort, item, feat, invocation, Weapon Mastery).

### 3. Fidélité bundle — 15-20 entrées de référence figées
- Fichier `tests/fixtures/srd-reference-entries.ts` — 20 entrées vérifiées contre le SRD 5.2.1 officiel (mix sorts/armes/classes/feats), valeurs gravées.
- Test : itère ces entrées, assert que `bundle.find(e.id)` égale la fixture EXACTEMENT.
- **Politique** : la vérification humaine SRD se fait UNE FOIS à l'ajout d'une entrée fixture (commit dédié `chore(content-truth): pin <slug>`), puis le test la maintient.

### 4. Calculs de règles — résultat chiffré
- Helpers `expectAC(character, expected)`, `expectAttackBonus(character, weaponId, expected)`, `expectSaveDC(character, ability, expected)`, etc.
- Cas paramétrés par build (cuir clouté + Dex 12 → CA = 13 ; cotte de mailles → CA = 16 ; ...).

### 5. Cohérence inter-écrans (wizard → fiche)
- Helper `submitWizardAndOpenSheet(wizardState)` → `{character, sheetRender}`.
- Pour chaque sous-choix L1, assert que le label FR posé au wizard est CELUI affiché sur la carte correspondante de la fiche (pas un autre slug du même enum).

### 6. Cas-limites — intersections
- Liste figée d'intersections SRD qui ont déjà cassé (et celles qui pourraient casser) :
  - Roublard Expertise sur compétence déjà maîtrisée par la classe → ×2, pas ×3.
  - Sort fourni par ascendance ET classe → 1 seule entrée sur la fiche.
  - Weapon Mastery sur arme non-maniée par la classe → interdit à la sélection.
  - Magicien préparant un sort non-inscrit → interdit.
  - Multi-classe Clerc 1 / Magicien 1 : `proficiencyBonus = 2` (total level), pas `2 + 2`.
- Chacun devient un test indépendant.

## Volet matriciel — Couverture combinatoire

### Axes
- Classe (×12 SRD).
- Ascendance (×9 SRD).
- Background (×N — à figer).
- Variants de campagne (×4 — feat L1 / flanking / slow healing / gritty).
- Sous-choix L1 obligatoires (par classe).

### Stratégie
- Combinatoire complète **niveau 1** → ~quelques milliers de personas (génération lazy).
- Snapshot léger par persona : `{characterValid: true/false, knownSpells.count, ac, totalLevel, errors[]}`.
- Détecte régressions globales d'un seul coup (« telle classe ne génère plus de personnage valide »).

### Non-objectifs
- PAS de snapshot visuel (pixel) — fragile et hors scope.
- PAS de couverture du level-up (plan 18).
