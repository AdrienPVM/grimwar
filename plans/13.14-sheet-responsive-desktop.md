# Plan 13.14 — Fiche + wizard pleinement responsive (mobile / tablet / desktop)

> **STATUT** : cadrage validé 2026-05-19 (cf. UAT 4a plan 13.9). À livrer après 13.12 (matrice) + 13.13 (CI) **selon position indicative** — Adrien peut remonter ce plan si besoin, ce n'est pas un verrou dur.

## Goal

La fiche de personnage (`/character/:id`) et le wizard (`/create`) restent **pleinement utilisables et soignés** sur toutes les tailles d'écran courantes — du smartphone (375 px) au desktop large (1920 px). Les 5 modes de la fiche (`combat` / `essence` / `magie` / `avoir` / `ame`) exploitent l'espace écran disponible (2 colonnes ou sidebar collapsible à partir de `lg:`, dense réparti par sections quand l'œil le supporte), pas un layout mobile étiré. L'identité visuelle (aurora background, glass panels, accents or, prototype) reste fidèle — c'est un travail d'exploitation de l'espace, pas une refonte. Le wizard, claim-rédigé « responsive mobile + desktop » au plan 05 mais jamais re-UAT après l'ajout des sous-choix d'ascendance (13.8) et de classe (13.9), reçoit un audit ciblé et des correctifs au besoin.

## Périmètre

### 1. Audit mode-par-mode (préalable)

Adrien fait l'audit **avant que ce plan ne commence vraiment**. Captures à 4 viewports — **375 / 768 / 1024 / 1440 px** (et idéalement 1920 px). Output : un tableau par mode listant les points qui crient (cards étirées, hero card hors viewport, modales floating sans contexte, navigation oubliée en haut, etc.). Ce tableau devient les steps d'implémentation.

### 2. Coquille fiche (`sheet-screen.tsx`)

- Mobile (< `lg:`) — inchangé, single column 460 px max, comportement actuel préservé.
- `lg:` (≥1024 px) — passage à un layout **2 colonnes** : sidebar collapsible à gauche (mode tabs + status strip + nav), contenu de mode à droite. Hero card devient sticky header en haut de la zone de contenu.
- `xl:` / `2xl:` (≥1280 / 1536 px) — densité ajustée (espace blanc qui respire, cards plus larges, sortie graphique de l'hexagram plus grande). Pas une 3e colonne — ce serait du gold-plating.

### 3. Modes de la fiche

| Mode | Stratégie desktop |
|---|---|
| Combat | Grille 2 cols à partir de `lg:` : HP + Attaques à gauche, conditions + style/mastery/race à droite. Battle HUD reste sticky en haut. |
| Essence | Hexagramme garde sa position centrale ; saves + skills passent en 2 cols à partir de `lg:` ; carte Ordre divin/primal (livré par 4b) prend sa place naturelle. |
| Magie | Magic circle reste centré ; spell list en grille à partir de `lg:` (2 cols ≥`lg:`, éventuellement 3 ≥`xl:`) ; sections Grimoire / Préparés (livré par 4b) prennent leurs places visuelles propres. |
| Avoir | Inventaire en grille 2 cols à partir de `lg:` ; bar poids reste en haut, coins en sidebar à droite à partir de `xl:`. |
| Âme | À cadrer — ce mode arrive en plan 20. Si livré au moment de 13.14, audit inclus ; sinon, ce plan documente ce qui est attendu. |

### 4. Wizard

- **Audit ciblé d'abord** : Adrien re-UAT `/create` à 1024 / 1440 / 1920 px. Le claim plan 05 « responsive mobile + desktop » est vrai en théorie mais 4 plans ont été livrés depuis (sous-choix ascendance 13.8, sous-choix classe 13.9, sorts ascendance cliquables 13.8b) sans audit desktop.
- **Si fuites trouvées** : correctifs ciblés inclus dans ce plan (pas un plan séparé).
- **Si l'audit révèle des choix structurels à revoir** (au-delà de l'ajustement mineur) : escalade vers un plan dédié, pas dans 13.14.

### 5. Tests de layout — structure DOM, pas pixel-diff

Tests vivants dans `tests/wizard-matrix/responsive/*` (qui s'appuie sur la matrice livrée par 13.12) :

- Pour chaque mode × 3 viewports (mobile/tablet/desktop), assertions **observables au sens DOM** :
  - Hero card visible sans scroll (`getBoundingClientRect().top >= 0 && bottom <= window.innerHeight`).
  - Aucune card overlap avec une autre (pas de chevauchement vertical de bbox).
  - Modal centered (left/right margins équilibrés à ± 1 px).
  - Sidebar collapsible disposée à gauche à `lg:` (left < hero card left).
- **Pas de pixel-diff Percy/Argos** — fragile, hors scope S1.

### 6. e2e responsive Playwright

Au moins 1 spec qui change la viewport au milieu d'un parcours utilisateur :

- Seed perso → ouvre `/character/:id` à 375 px → bascule à 1440 px → vérifie que la fiche reste cohérente (hero visible, mode tabs accessibles, mode actuel n'a pas disparu, navigation entre modes fonctionne).

### 7. Tokens design system

- Vérifier qu'aucun magic number ne traîne dans le code livré.
- Breakpoints viennent uniquement de Tailwind ou de tokens documentés.
- Si un nouveau token visuel est requis (espacement de sidebar, hauteur de hero sticky), il passe par `docs/DESIGN-SYSTEM.md`.
