# Plan 13.6 — LibraryScreen + nav shell

> **Exécuté 2026-05-16. UAT navigateur par Adrien en attente.**

## Goal

`/` cesse d'être un cul-de-sac. L'utilisateur qui ouvre GrimWar arrive sur une **LibraryScreen** réelle : liste des personnages qu'il possède (lus dans Firestore `users/{uid}/characters/`), card par perso avec emblème HP + nom + sous-titre multi-class + statut alive/dead, CTA « Créer un personnage » visible. Un nav shell minimal — header sticky or/bronze fidèle au prototype — persiste entre `/` et `/character/:id`, et permet le retour explicite à la library depuis la fiche sans toucher l'URL. État vide soigné pour les premiers utilisateurs (0 perso). État chargement sans flash blanc. Lyralei hardcodée éliminée du code de prod.
