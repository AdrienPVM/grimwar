# I18n

French first. English added in Sprint 5. The architecture supports adding more languages without refactor.

## Two distinct concerns

| Concern | Source | Resolution |
|---|---|---|
| **UI strings** (buttons, labels, errors, tooltips) | `src/shared/lib/i18n.ts` flat key map | `t('key', vars?)` |
| **Entity prose** (spell names, monster descriptions, class features) | JSON files with `{ fr, en? }` shapes | `localize(value, locale)` |

Both resolve via the same `useLocale()` hook providing the current locale.

## UI strings

All UI strings have a key. Keys use dot-notation grouped by feature:

```ts
// src/shared/lib/i18n.ts
export const STRINGS = {
  fr: {
    'auth.signIn': 'Se connecter',
    'auth.signInWithGoogle': 'Continuer avec Google',
    'auth.signInWithEmail': "Continuer avec l'email",
    'campaign.create.title': 'Nouvelle campagne',
    'campaign.create.namePlaceholder': 'Le nom de votre campagne',
    'campaign.invite.codeLabel': "Code d'invitation",
    'sheet.combat.hp': 'Points de vie',
    'sheet.combat.deathSaves.title': 'Sauvegardes de mort',
    'common.cancel': 'Annuler',
    'common.confirm': 'Confirmer',
    // …
  },
  en: {
    // populated in S5
  },
} satisfies Record<Locale, Record<string, string>>;

export function t(key: string, vars?: Record<string, string | number>): string {
  const locale = useLocaleStore.getState().locale;
  let str = STRINGS[locale]?.[key] ?? STRINGS.fr[key] ?? key;
  if (vars) {
    for (const [k, v] of Object.entries(vars)) {
      str = str.replace(`{${k}}`, String(v));
    }
  }
  return str;
}
```

**Rules**:
- NEVER hardcode user-facing strings in components.
- Add new keys in alphabetical order within feature groups.
- Test that adding a new key doesn't break existing tests.
- Keys are stable: renaming a key is a migration.

## Entity prose

All content entities use the `I18n` shape:

```ts
type I18n = { fr: string; en?: string };
```

Examples:

```ts
// spells.json
{
  id: 'fireball',
  name: { fr: 'Boule de feu', en: 'Fireball' },
  description: { fr: 'Une vive lueur jaillit…', en: 'A bright streak flashes…' },
  castingTime: { fr: '1 action' },
  // …
}
```

In components:

```tsx
import { localize } from '@/shared/lib/i18n';
const locale = useLocale();
<h3>{localize(spell.name, locale)}</h3>
<p>{localize(spell.description, locale)}</p>
```

**Always FR fallback**: `localize({ fr: 'x' }, 'en')` returns `'x'`.

## Locale source of truth

- User's preferred locale → `users/{uid}.locale`
- Default for anonymous users → browser `navigator.language` starts-with `'fr'` ? `'fr'` : `'fr'` (yes, FR even for non-FR for v0.x; EN as toggle in S5)
- Zustand `useLocaleStore` holds the active locale
- On user login, sync to Firestore
- Reactive: changing locale re-renders the whole tree

## Locale switcher

Settings screen has a Locale switcher (S5). Pre-S5, the locale is hard-coded to `'fr'` everywhere.

## Date / number formatting

Use `Intl.DateTimeFormat` and `Intl.NumberFormat` keyed off the current locale. Don't hardcode formats.

Helpers in `src/shared/lib/format.ts`:

```ts
formatDate(date: Date, locale: Locale, options?: Intl.DateTimeFormatOptions): string
formatNumber(value: number, locale: Locale, options?: Intl.NumberFormatOptions): string
formatRelativeTime(date: Date, locale: Locale): string
```

## Plural forms

Standard `Intl.PluralRules` for cardinal/ordinal. Keep keys with `_one`, `_other` suffixes (Crowdin/i18next convention) for future migration:

```
'character.spells.count_one': '{count} sort'
'character.spells.count_other': '{count} sorts'
```

Helper `tPlural('character.spells.count', count)` picks the right key.

## What NOT to do

- Don't translate keys (always English snake/dot-case)
- Don't store locale-specific HTML
- Don't use string concatenation: `'You have ' + count + ' spells'` — use placeholder interpolation
- Don't put translation logic in components — always go through `t()` / `localize()`
